# Course_design_REVA
Syllabus Unit & Activity Name,Duration,Step-by-Step Execution Plan,Recommended Tools,Grading Rubric & Breakdown
1. Introduction & Data Modeling📊 Real-World ERD Reverse Engineering,1 Week(Out-of-class prep + 1 Lab Session),"1. Divide class into teams of 3–4 students.2. Assign a popular app feature (e.g., Spotify's playlist system, Uber's ride-matching, or Airbnb's reviews).3. Teams identify all core Entities, Attributes, Primary Keys, and Foreign Keys.4. Map relationships and explicitly define cardinalities (1:1, 1:M, M:N).5. Construct an Extended ER (EER) diagram including specialization/generalization hierarchies.","Draw.io, Lucidchart, or Miro",Total: 10 Marks• Correctness of Entities/Attributes (3M)• Relationship/Cardinality Mapping (3M)• Normalization Awareness in Design (2M)• Presentation & Q&A Defense (2M)
2. Relational Model & Normalization🧼 The Schema Normalization Clinic,1 Lab Session(2–3 Hours),"1. Distribute a massive, deliberately un-normalized dataset (e.g., a ""Hospital Management"" Excel sheet matching doctors, patients, billing, and history in a single row).2. Task 1: Students must manually trigger and document an Insertion, Deletion, and Update Anomaly.3. Task 2: Extract and list all true Functional Dependencies (X→Y).4. Task 3: Systematically decompose the table step-by-step from 1NF to 2NF, and finally into 3NF or BCNF.","MS Excel / Google Sheets, Pen & Paper",Total: 10 Marks• Anomalies Identification (2M)• Accurate Functional Dependency List (3M)• Flawless Decomposition to 3NF/BCNF (Lossless & Dependency Preserving) (5M)
3. Structured Query Language (SQL)🕵️‍♂️ Live SQL Detective Mystery,1 Lab Session(2 Hours),"1. Load an SQLite/PostgreSQL database containing a fictional city's crime logs, witness accounts, gym check-ins, and driver records.2. Provide a starting prompt: ""A crime occurred on Jan 15. The suspect is a male with a gold gym membership driving a car with a plate containing 'H42'.""3. Students must string together complex SELECT queries utilizing multi-table JOINs, subqueries, wildcards, and aggregations to hunt down the suspect.","DBeaver, SQLite Browser, or SQL Murder Mystery repo","Total: 10 Marks• Successfully finding the culprit (4M)• Query Efficiency (Avoided nested loop hell, used clean JOINs) (4M)• Code Cleanliness & Indentation/Comments (2M)"
"4. Indexing & Query Optimization⚡ The ""Slow Query"" Optimization Lab",1 Lab Session(2 Hours),"1. Pre-populate a database with 1+ million rows of mock e-commerce transactions (orders, customers, products).2. Provide 3 poorly written, slow queries (e.g., heavy wildcard text searches or unindexed WHERE string matches).3. Students run EXPLAIN ANALYZE to map out the execution cost (e.g., locating slow Sequential Scans).4. Students create strategic B-Tree or Hash indexes, optimize the query syntax, re-run EXPLAIN ANALYZE, and document the performance jump.","PostgreSQL / pgAdmin, MySQL CLI",Total: 10 Marks• Interpretation of initial EXPLAIN plan (2M)• Strategic Index Selection (Single vs. Composite) (4M)• Percentage of execution time reduced (Performance gain) (4M)
5. Transaction & Concurrency Control🔄 Transaction Simulation Game,1 Lab Session(2 Hours),"1. Give students a basic banking script containing user accounts and balances.2. Spin up two concurrent terminal sessions (Transaction A & B) interacting with the exact same database row.3. Guide students to enter interleaved SQL code manually to intentionally induce a Dirty Read, Non-Repeatable Read, and a Deadlock.4. Students then apply SET TRANSACTION ISOLATION LEVEL configurations to observe how the engine handles lock conflicts.",Relational DB engine (MySQL/Postgres) using dual terminal screens,Total: 10 Marks• Successful manual replication of concurrency anomalies (4M)• Correct resolution using isolation levels (4M)• Post-lab analysis report on performance vs. integrity trade-offs (2M)
6. Advanced Topics (NoSQL/NewSQL)🌐 Polyglot Persistence Challenge,2 Weeks(Mini-Project / Take-home assignment),1. Students design a hybrid database architecture for a modern e-commerce application.2. SQL Layer: Implement user authentication profiles and the financial ledger in a relational database to ensure strict ACID compliance.3. NoSQL Layer: Implement the dynamic product catalog tags or real-time user notification streams using a document store.4. Write a minimal backend script to query both data layers and display the merged data on a unified dashboard.,"PostgreSQL (SQL), MongoDB (NoSQL), Node.js or Python (Flask/FastAPI)",Total: 20 Marks• Justification of structural data split (5M)• Correct Schema Design in both environments (5M)• Backend connectivity & cross-database rendering (5M)• Error handling & code architecture (5M)
